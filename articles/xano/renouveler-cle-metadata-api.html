Renouveler sa clé Metadata API Xano expirée

Dans cet article, je vais expliquer comment résoudre le problème d'une clé Metadata API Xano expirée et générer une nouvelle clé sans date d'expiration pour une utilisation stable et permanente.

🏷️ Catégorie: Xano

🎯 Niveau: Débutant

🔍 Mots-clés: Metadata API, Clé API, Authentication, Developer Access, Sécurité

📅 Mise à jour: 19/10/2025

⏱️ Temps passé: 20 min

⚙️ Stack: Xano

✅ TL;DR : Si ta clé Metadata API Xano est expirée, génère-en une nouvelle sans date d'expiration depuis la page compte Xano. La section "Developer Access" est dépréciée et ne doit plus être utilisée.

🤔 Problème rencontré

🛑 Situation initiale :

Tu cherches à obtenir une clé Xano pour l'utiliser avec une intégration externe (comme Supabase ou une autre API externe). En accédant à ta page compte Xano, tu remarques un message indiquant que l'accès développeur (Developer Access) est déprécié. Cela crée une confusion sur l'endroit où récupérer une clé valide.

💡 Symptômes observés :

  • Les appels API nécessitant la clé Metadata échouent
  • Message d'erreur d'authentification ou d'autorisation
  • La section "Developer Access" affiche un avertissement de dépréciation
  • La clé existante ne fonctionne plus

🔍 Explication simplifiée

Qu'est-ce qu'une clé Metadata API ?

La clé Metadata API de Xano est une clé d'authentification spéciale qui permet d'interagir avec les API de métadonnées de Xano. Contrairement aux clés d'API classiques utilisées pour tes endpoints, cette clé sert à manipuler la structure de ton backend (tables, champs, workflows, etc.).

🧠 Analogie :

Imagine que ton projet Xano est un immeuble :

  • Les clés d'API classiques permettent d'entrer dans les appartements (accéder aux données)
  • La clé Metadata API est la clé du concierge qui permet de modifier la structure du bâtiment lui-même (ajouter des pièces, changer la plomberie, etc.)
⚠️ Attention
La clé Metadata API est très puissante. Elle permet de modifier toute la structure de ton backend Xano. Ne la partage jamais publiquement et ne la commit pas dans un dépôt Git public !

🛠️ Solution : Générer une nouvelle clé

Étape 1️⃣ : Accéder à la page compte Xano

Connecte-toi à Xano et accède directement à ta page de compte en mode master :

🔗 https://app.xano.com/account?mode=master

Étape 2️⃣ : Localiser la section Metadata API Key

Sur la page de ton compte, cherche la section "Metadata API Key". Cette section est distincte de l'ancienne section "Developer Access" qui est maintenant dépréciée.

✨ Astuce
Si tu vois un message concernant "Developer Access is deprecated", ignore-le. Ce n'est plus la méthode recommandée pour obtenir une clé Metadata.

Étape 3️⃣ : Générer une nouvelle clé sans expiration

Dans la section Metadata API Key :

  1. Clique sur le bouton pour générer une nouvelle clé
  2. Important : Coche l'option pour créer une clé SANS date d'expiration
  3. Valide la génération
  4. Copie immédiatement la clé générée et stocke-la en sécurité
⚠️ Attention
La clé ne sera affichée qu'une seule fois. Si tu la perds, tu devras en générer une nouvelle. Stocke-la dans un gestionnaire de mots de passe ou un fichier d'environnement sécurisé.

Étape 4️⃣ : Remplacer l'ancienne clé

Partout où tu utilisais l'ancienne clé expirée, remplace-la par la nouvelle :

  • Dans tes variables d'environnement
  • Dans tes fichiers de configuration
  • Dans tes intégrations externes (Supabase, Make, Zapier, etc.)

🧰 Exemple concret d'utilisation

Contexte : Tu utilises la clé Metadata API pour synchroniser automatiquement tes tables Xano avec Supabase.

// Avant (clé expirée)
const XANO_METADATA_KEY = "xmu_old_expired_key_12345";

// Après (nouvelle clé sans expiration)
const XANO_METADATA_KEY = "xmu_new_permanent_key_67890";

// Utilisation dans une requête
fetch('https://x8ki-letl-twmt.n7.xano.io/api:metadata/table/list', {
  headers: {
    'Authorization': `Bearer ${XANO_METADATA_KEY}`,
    'Content-Type': 'application/json'
  }
})
.then(response => response.json())
.then(data => console.log('Tables:', data));
🔬 Pour aller plus loin
La Metadata API de Xano permet de faire bien plus que simplement lister les tables. Tu peux créer des tables, modifier des champs, gérer des relations, et même automatiser la création de ton backend via du code. Explore la documentation officielle de Xano pour découvrir toutes les possibilités.

📌 À retenir

📌 À retenir :
  • La section "Developer Access" de Xano est dépréciée - ne l'utilise plus
  • Génère ta clé Metadata API depuis la page compte Xano (mode master)
  • Crée toujours une clé sans date d'expiration pour éviter les interruptions
  • Stocke ta clé de manière sécurisée (jamais en clair dans ton code)
  • La clé Metadata API est très puissante - protège-la comme un mot de passe

🧠 Quiz rapide

Question : Quelle est la différence entre une clé d'API Xano classique et une clé Metadata API ?

✅ Voir la réponse

Une clé d'API classique permet d'accéder aux données via les endpoints que tu as créés (lecture/écriture dans les tables). La clé Metadata API permet de manipuler la structure de ton backend Xano lui-même (créer/modifier des tables, des champs, des workflows, etc.).
