Base de données Xano

Création de tables

Les tables dans Xano sont l'équivalent des tables dans une base de données relationnelle traditionnelle. Chaque table stocke un type spécifique de données.

Types de champs disponibles

Champs basiques

  • Text : Chaînes de caractères courtes
  • Integer : Nombres entiers
  • Float : Nombres décimaux
  • Boolean : Vrai/Faux
  • Timestamp : Date et heure

Champs avancés

  • JSON : Objets JSON structurés
  • File : Upload de fichiers (images, documents)
  • Enum : Liste de valeurs prédéfinies
  • Email : Validation automatique d'email
  • URL : Validation d'URLs

Champs calculés

  • Auto-increment : ID unique auto-incrémenté
  • Created_at : Date de création automatique
  • Updated_at : Date de dernière modification

Relations entre tables

One-to-One (1:1)

Un enregistrement dans la table A est lié à exactement un enregistrement dans la table B. Exemple : Un utilisateur a un profil.

One-to-Many (1:N)

Un enregistrement dans la table A peut être lié à plusieurs enregistrements dans la table B. Exemple : Un utilisateur peut avoir plusieurs commandes.

Many-to-Many (N:N)

Plusieurs enregistrements de la table A peuvent être liés à plusieurs enregistrements de la table B. Exemple : Un produit peut avoir plusieurs catégories, une catégorie peut avoir plusieurs produits.

Exemple pratique : Base de données e-commerce

Voici comment structurer une base de données simple pour un site e-commerce :

Table "users"

  • id (auto-increment)
  • email (email)
  • name (text)
  • created_at (timestamp)

Table "products"

  • id (auto-increment)
  • name (text)
  • description (text)
  • price (float)
  • image (file)
  • stock (integer)

Table "orders"

  • id (auto-increment)
  • user_id (relation vers users)
  • total (float)
  • status (enum: pending, paid, shipped, delivered)
  • created_at (timestamp)

Table "order_items"

  • id (auto-increment)
  • order_id (relation vers orders)
  • product_id (relation vers products)
  • quantity (integer)
  • price (float)

Bonnes pratiques

  1. Nommage cohérent : Utilisez des noms descriptifs et en anglais
  2. Indexes : Ajoutez des index sur les champs fréquemment recherchés
  3. Validation : Définissez des règles de validation pour chaque champ
  4. Valeurs par défaut : Définissez des valeurs par défaut quand c'est pertinent
  5. Documentation : Ajoutez des descriptions à vos tables et champs
