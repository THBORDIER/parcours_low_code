À propos de ce site et ma méthode de travail

Comment je transforme mes blocages en apprentissage partagé

Qui suis-je ?

Je m'appelle Thomas, et je suis actuellement étudiant sur le parcours Développeur Low-Code chez OpenClassrooms.

Point important : Je ne suis pas développeur de formation. Je découvre le développement web, les API, les bases de données et l'architecture applicative au fur et à mesure de ma formation. Ce site est le reflet de mon apprentissage progressif.

Objectif de ce site : Documenter mes apprentissages, résolutions de problèmes, et partager mes découvertes avec mes pairs de formation – tout en me créant une base de mémos personnels.

Pourquoi cette méthode de documentation ?

Au cours de ma formation, j'ai rencontré de nombreux blocages techniques. Plutôt que de simplement résoudre le problème et passer à autre chose, j'ai décidé de capitaliser sur chaque difficulté en la documentant de manière claire et structurée.

Les bénéfices de cette approche

  • Pour moi : Consolider mes apprentissages, créer une base de connaissances réutilisable
  • Pour mes pairs : Partager des solutions concrètes à des problèmes réels
  • Pour ma progression : Transformer chaque erreur en opportunité d'apprentissage
  • Pour la communauté : Contribuer à l'écosystème no-code francophone

Mon workflow de travail avec l'IA

Je ne travaille pas seul. J'utilise plusieurs modèles d'intelligence artificielle pour m'assister à différentes étapes de mon processus d'apprentissage et de documentation.

Étape Outil IA Rôle
1. Résolution de problème ChatGPT Comprendre le blocage, explorer les solutions, débugger
2. Documentation structurée Perplexity Synthétiser le problème, les solutions, et le chemin parcouru
3. Intégration HTML/CSS Claude Code Transformer la documentation en article web formaté
4. Versioning GitHub Sauvegarder et versionner les modifications
5. Déploiement Vercel Déploiement automatique en production

Détail du processus

Étape 1 : Résolution avec ChatGPT

Lorsque je rencontre un blocage (workflow WeWeb qui ne fonctionne pas, erreur API Xano, etc.), je commence par dialoguer avec ChatGPT pour :

  • Comprendre la nature du problème
  • Identifier les causes possibles
  • Tester différentes solutions
  • Débugger pas à pas

Étape 2 : Documentation avec Perplexity

Une fois le problème résolu, je demande à Perplexity de structurer la conversation en :

  • Problème initial : Description claire du blocage rencontré
  • Solutions explorées : Les différentes approches testées (qui ont échoué et pourquoi)
  • Solution retenue : La méthode qui a fonctionné
  • Apprentissages clés : Ce qu'il faut retenir pour éviter ce problème à l'avenir

Étape 3 : Intégration avec Claude Code

Je fournis le résumé structuré à Claude Code qui :

  • Transforme le contenu en HTML sémantique
  • Applique le style CSS du site (respecte la charte graphique)
  • Ajoute des exemples de code, tableaux, et listes formatés
  • Insère l'article dans l'arborescence du site

Étape 4 : Versioning avec GitHub

Claude Code commit automatiquement les modifications sur GitHub, ce qui me permet de :

  • Garder un historique de toutes les versions
  • Revenir en arrière si nécessaire
  • Collaborer si besoin (open source possible)

Étape 5 : Déploiement automatique avec Vercel

Dès que je push sur GitHub, Vercel détecte les modifications et déploie automatiquement la nouvelle version du site en production. Zéro manipulation manuelle.

Avantage de ce workflow : Chaque résolution de problème devient immédiatement un article publié et accessible à tous, sans effort supplémentaire de ma part.

Méthodologie de documentation

Quand est-ce que je documente ?

Je documente uniquement les situations où :

  • J'ai rencontré un blocage significatif (pas les questions triviales)
  • La solution n'était pas évidente
  • Le chemin pour résoudre le problème m'a appris quelque chose d'important
  • Mes pairs pourraient rencontrer le même problème

Structure type d'un article

  1. Problème rencontré : Description claire du blocage
  2. Contexte : Environnement technique (WeWeb, Xano, versions, etc.)
  3. Solutions explorées : Ce qui n'a pas fonctionné
  4. Solution retenue : La méthode qui a résolu le problème
  5. Explication technique : Pourquoi ça fonctionne
  6. Points clés à retenir : Les apprentissages essentiels

La communauté no-code francophone

Comment utiliser ce site ?

Navigation par thème

Les articles sont organisés en 5 grandes catégories :

  • WeWeb : Frontend no-code, composants, workflows, déploiement
  • Xano : Backend no-code, base de données, API, fonction stacks
  • API & Backend : Intégrations, REST API, authentification
  • Bonnes pratiques : Architecture, conventions, optimisations
  • Notes diverses : Ressources, outils, astuces de productivité

Recherche par problème

Si vous rencontrez un problème précis, utilisez la fonction de recherche de votre navigateur (Ctrl+F ou Cmd+F) pour rechercher des mots-clés dans les titres d'articles.

Consulter les exemples de code

Chaque article contenant du code inclut des exemples formatés et commentés. Vous pouvez les copier directement dans vos projets.
