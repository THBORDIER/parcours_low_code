Astuces de productivité

Raccourcis clavier WeWeb

Navigation

Ctrl/Cmd + P : Quick search (pages, composants)
Ctrl/Cmd + S : Sauvegarder
Ctrl/Cmd + Z : Annuler
Ctrl/Cmd + Shift + Z : Refaire
Ctrl/Cmd + D : Dupliquer l'élément sélectionné

Édition

Ctrl/Cmd + C : Copier
Ctrl/Cmd + V : Coller
Ctrl/Cmd + X : Couper
Delete/Backspace : Supprimer l'élément
Arrow keys : Déplacer l'élément
Shift + Arrow keys : Déplacer par incréments plus grands

Vue et preview

Ctrl/Cmd + E : Mode édition/preview
Ctrl/Cmd + / : Afficher/masquer le panneau de propriétés

Workflows efficaces

1. Planification avant exécution

  • Wireframes : Dessinez d'abord sur papier ou Figma
  • User flows : Mappez les parcours utilisateur
  • Data model : Schématisez votre base de données avant de créer les tables

2. Composants réutilisables

Ne dupliquez jamais le même bloc 3 fois. À la place :

  1. Créez un composant personnalisé
  2. Rendez-le configurable avec des propriétés
  3. Réutilisez-le partout

3. Templates de page

  • Créez une page "template" avec header + footer
  • Dupliquez-la pour chaque nouvelle page
  • Gain de temps considérable

Organisation du travail

Méthode Pomodoro adaptée

25 min : Focus sur une tâche précise (ex: créer un endpoint)
5 min : Pause
Après 4 pomodoros : Pause de 15-20 min

Priorisation des tâches

Matrice Eisenhower

  • Urgent + Important : Bugs critiques, deadline proche
  • Important mais pas urgent : Optimisations, refactoring
  • Urgent mais pas important : Petites demandes, à déléguer si possible
  • Ni urgent ni important : À éliminer

Gestion des tâches

2. [ ] Intégrer dans WeWeb
3. [ ] Tester le flow complet

Backlog :
- Ajouter pagination
- Optimiser images
- Documentation API

Astuces Xano

1. Dupliquer les endpoints similaires

Plutôt que recréer de zéro, dupliquez un endpoint existant et modifiez-le.

2. Fonctions réutilisables

Créez des "Custom Functions" pour la logique répétitive :

  • send_email
  • validate_input
  • format_date
  • calculate_total

3. Utilisez les filtres dans les queries

Plutôt que charger tous les users et filtrer après,
utilisez directement les filtres dans "Query All Records"

4. Testez avec le debugger intégré

  • Utilisez "Run & Debug" pour tester vos endpoints
  • Vérifiez chaque étape du Function Stack
  • Examinez les variables intermédiaires

Astuces WeWeb

1. Variables pour éviter la répétition

❌ Mauvais :
Répéter la même formule complexe 10 fois

✅ Bon :
Stocker dans une variable et la réutiliser

2. Utilisez les styles globaux

  • Définissez vos couleurs principales une fois
  • Créez des classes CSS réutilisables
  • Gain de temps et cohérence visuelle

3. Workflows conditionnels plutôt que multiples workflows

✅ Un workflow avec conditions :
IF user.role === 'admin'
  THEN redirect to /admin
  ELSE redirect to /dashboard

4. Mode debug

  • Activez la console du navigateur
  • Utilisez console.log dans vos formules
  • Tracez l'exécution de vos workflows

Outils de productivité

Snippets de code

Créez une bibliothèque de snippets réutilisables :

  • Formules de validation
  • Transformations de données courantes
  • Structures de workflows récurrents

Documentation as you go

  • Ajoutez des commentaires dans Xano Function Stacks
  • Nommez clairement vos variables et composants
  • Maintenez un changelog des modifications

Templates de projet

Créez un template de base avec :

  • Authentification pré-configurée
  • Pages standards (login, signup, profile)
  • Composants communs (header, footer, buttons)
  • Structure de données de base

Éviter les pièges courants

1. Ne pas tester au fur et à mesure

Testez chaque fonctionnalité dès qu'elle est créée, pas à la fin.

2. Négliger la gestion d'erreurs

Ajoutez toujours un "On Error" workflow pour gérer les cas d'échec.

3. Optimiser trop tôt

Faites d'abord fonctionner, optimisez ensuite. "Premature optimization is the root of all evil."

4. Ne pas faire de pauses

Les pauses régulières augmentent la productivité et la créativité.

Checklist quotidienne

Début de journée

  • Définir 3 objectifs principaux
  • Vérifier les messages/notifications
  • Prioriser les tâches

Pendant le travail

  • Focus sur une tâche à la fois
  • Tester au fur et à mesure
  • Commiter régulièrement (si Git)
  • Documenter les décisions importantes

Fin de journée

  • Vérifier que tout fonctionne
  • Noter ce qui reste à faire
  • Sauvegarder le travail
  • Planifier le lendemain
