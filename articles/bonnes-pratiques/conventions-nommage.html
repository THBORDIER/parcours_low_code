Conventions de nommage

Pourquoi des conventions de nommage ?

Des conventions claires et cohérentes facilitent :

  • La compréhension du code par l'équipe
  • La maintenance à long terme
  • La recherche d'éléments spécifiques
  • L'onboarding de nouveaux membres

Conventions générales

Casse (Case)

Contexte Convention Exemple
Tables Xano snake_case order_items
Champs Xano snake_case first_name
Endpoints Xano kebab-case /order-items
Pages WeWeb PascalCase ProductsList
Variables WeWeb camelCase currentUser
Composants WeWeb PascalCase CardProduct

Préfixes utiles

  • is_ / has_ pour les booléens : is_active, has_permission
  • get_ pour les fonctions de lecture : get_user_by_email
  • create_ pour les créations : create_order
  • update_ pour les modifications : update_user_profile
  • delete_ pour les suppressions : delete_product

Longueur des noms

  • Trop court : u, p, d → manque de clarté
  • Équilibré : user, product, currentUser
  • Trop long : theCurrentlyAuthenticatedUserInformation → difficile à lire

Exceptions acceptables

  • Conventions établies : id, url, api
  • Abréviations courantes : max, min, avg
  • Standards métier : sku, vat, ean

Application des conventions

En équipe

  1. Définir un document de conventions
  2. Le partager avec toute l'équipe
  3. Le revoir régulièrement
  4. Faire des code reviews pour vérifier le respect

En solo

  1. Choisir des conventions dès le début
  2. Les documenter dans le README
  3. S'y tenir même quand on est seul
  4. Refactoriser si nécessaire pour maintenir la cohérence
