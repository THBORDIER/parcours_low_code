Comprendre l'arborescence d'une page WeWeb : exemple de la page Mon Compte

Dans cet article, nous découvrons la structure complète d'une page construite avec WeWeb. L'objectif est de comprendre la logique d'imbrication des éléments et leur rôle dans l'interface utilisateur. Cette organisation hiérarchique est essentielle pour maintenir une page claire, responsive et facilement maintenable.

Figure 1: Structure hiérarchique d'une page Mon Compte dans WeWeb

Niveau 1 : Section principale

Tout commence par un composant parent appelé Section. C'est le conteneur global qui définit la zone de la page visible par l'utilisateur.

Enfants directs :

  • main
  • footer

Ces deux éléments représentent respectivement le contenu central de la page et le bas de page.

Niveau 2 : le contenu principal (main)

À l'intérieur de main, on retrouve deux blocs structurants :

  • cl_navbar : correspond à la barre de navigation supérieure.
  • div_principal : c'est ici que se trouve tout le contenu du profil utilisateur.

Niveau 3 : Structure interne de div_principal

ctn_principal

Il agit comme un container principal pour organiser les données du profil en sections logiques.

Sous-éléments de ctn_principal :

div_photo

  • Contient l'image de profil (nommée profile picture).
  • Permet de personnaliser l'espace utilisateur.

div_vos infos

C'est le bloc regroupant toutes les informations personnelles. Cette division est ensuite organisée en deux sous-catégories :

  • div_titre : affiche le titre ou en-tête ("Vos informations" par exemple).
  • div_details : contient les champs de saisie du profil.

Détail des champs dans div_details

Chaque information personnelle est présentée sous forme de couple Label + Input, respectant une logique UX claire :

Élément Rôle
Label_adresse mail Texte d'indication ("Adresse email")
Input_prenom Champ de saisie du prénom
Label_nom Libellé pour le nom de famille
Input_nom Champ pour saisir le nom
Label_prenom Libellé pour le prénom
Input_adresse mail Champ de saisie pour l'adresse email
Div Un sous-conteneur (probablement pour regrouper certains champs)
Input_telephone Champ de saisie du numéro de téléphone

Chacun de ces éléments est imbriqué dans le div_details, lui-même imbriqué dans div_vos infos, ce qui permet :

  • Une organisation claire
  • Un design responsive
  • Une séparation logique entre affichage, contenu et interaction utilisateur

Niveau final : footer

Placée en bas de page, cette zone sert généralement à afficher :

  • Mentions légales
  • Liens de navigation secondaires
  • Copyright

Logique d'imbrication résumée

Section
 ├── main
 │    ├── cl_navbar
 │    └── div_principal
 │         └── ctn_principal
 │              ├── div_photo
 │              │     └── profile picture
 │              └── div_vos infos
 │                    ├── div_titre
 │                    └── div_details
 │                          ├── Label + Input (Nom)
 │                          ├── Label + Input (Prénom)
 │                          ├── Label + Input (Email)
 │                          └── Input téléphone
 └── footer

Conclusion

Cette structure reflète une organisation professionnelle et modulaire :

  • Chaque bloc a une fonction spécifique : cela facilite la maintenance et la compréhension du code
  • L'information est regroupée par pertinence : les éléments liés sont imbriqués ensemble
  • L'imbrication favorise la réutilisation et l'adaptabilité : chaque composant peut être modifié indépendamment
